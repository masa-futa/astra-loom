# 星空UI デザイン・実装計画

## 📋 プロジェクト概要

美しく、かつ天文学的に正確な星空表示を実現するiPadアプリのUI設計。

**目標**: 科学的正確性と視覚的な美しさの両立

---

## 🎨 デザインフィロソフィー

### コンセプト検討

#### オプションA: リアリスティック・アプローチ
**特徴:**
- 実際の天候データと連携
- 時刻・季節による背景変化
- 月齢による明るさ調整
- 光害の表現

**メリット:**
- ✅ 教育的価値が高い

**デメリット:**

**必要なAPI:**
- OpenWeatherMap（天候）
- 月齢計算（既にKMPで実装可能）
- 光害マップ（オプション）

#### オプションB: ファンタジー・アプローチ
**特徴:**
- 常に美しい星空
- 芸術的なグラデーション
- 幻想的なエフェクト
- パーティクルアニメーション

**メリット:**
- ✅ 常に美しい体験
- ✅ 実装がシンプル
- ✅ パフォーマンスが良い
- ✅ オフラインで完全動作

**デメリット:**
- ❌ 教育的価値が低い

---

#### ✨ 推奨: ハイブリッド・アプローチ

**コンセプト:**
「リアルな星の配置 × 美しいビジュアル表現」

**実装:**
1. **星の配置**: 実際の天文データに基づく（既にKMPで実装済み）
2. **背景**: 時刻に応じた幻想的なグラデーション
3. **オプション**: リアル/ファンタジーモード切り替え

**時刻による背景変化:**
```
夕暮れ（18:00-19:00）: オレンジ→紫のグラデーション
宵の口（19:00-21:00）: 紫→深い青
深夜（21:00-04:00）: 深い青→黒
明け方（04:00-06:00）: 黒→紫→オレンジ
```

**天候モード（オプション機能）:**
- 設定で「リアル天候」ON/OFF
- ONの場合のみ天候API使用
- デフォルトは常に晴れ

## 🌟 星の表示設計

### データ拡充計画

**現状:**
- 25個の明るい星（magnitude < 4.0）

**目標:**
- 100-200個の星（magnitude < 5.0）
- 天の川の表現用に数千の暗い星（magnitude < 6.5）のデータ

**データソース候補:**
1. Hipparcos Catalog（11万個の星）
2. Yale Bright Star Catalog（9,110個）
3. 埋め込みデータとして厳選

**推奨アプローチ:**
```
- Phase 1: 100個の明るい星（mag < 4.5）を埋め込み
- Phase 2: 500個の星（mag < 5.5）
- Phase 3: 天の川用に3000個の暗い星（mag < 6.5）
```

### 星の視覚表現

**サイズ:**
```swift
// 等級に応じたサイズ計算
baseSize = 8.0
size = baseSize * pow(2.0, (1.5 - magnitude) / 2.0)

例:
magnitude -1.5 (Sirius): 16pt
magnitude  0.0: 12pt
magnitude  2.0: 8pt
magnitude  4.0: 4pt
magnitude  6.0: 2pt
```

**色:**
```swift
スペクトル型による色分け:
- O, B型: 青白 (RGB: 155, 176, 255)
- A型: 白 (RGB: 202, 215, 255)
- F型: 黄白 (RGB: 248, 247, 255)
- G型: 黄 (RGB: 255, 244, 234)
- K型: オレンジ (RGB: 255, 210, 161)
- M型: 赤 (RGB: 255, 204, 111)
```

**輝き効果:**
- Bloom/Glow エフェクト
- 明るい星ほど強い輝き
- アニメーション（微細な明滅）

## 🎨 背景グラデーション設計

### デフォルト（ファンタジーモード）

**グラデーション構成:**
```swift
下部（地平線）:
  - Color(hex: "1a1a2e") // 深い紺
  - Color(hex: "16213e") // 濃い青

中央:
  - Color(hex: "0f3460") // ミッドナイトブルー

上部（天頂）:
  - Color(hex: "0a1929") // ほぼ黒
  - Color(hex: "000000") // 純粋な黒
```

**時刻による調整:**
```swift
夕暮れ:
  下部に追加: オレンジ(255, 140, 0, 0.3)

明け方:
  下部に追加: ピンク(255, 105, 180, 0.2)

深夜:
  全体的に暗く
```

### リアルモード（オプション）

**太陽高度による計算:**
```
太陽が地平線下 -18度以下: 完全な夜（上記グラデーション）
-18度 ~ -12度: 天文薄明（明るさ+20%）
-12度 ~ -6度: 航海薄明（明るさ+40%）
-6度 ~ 0度: 市民薄明（明るさ+60%、オレンジ追加）
```

## 🔧 技術実装

### 描画技術の選択

#### オプション1: SwiftUI Canvas
**メリット:**
- ✅ SwiftUIネイティブ
- ✅ 実装が簡単
- ✅ デバッグしやすい

**デメリット:**
- ❌ パフォーマンスに限界
- ❌ 複雑なエフェクトが難しい

**推奨用途:** Phase 1（100個の星まで）

#### オプション2: Metal
**メリット:**
- ✅ 高パフォーマンス
- ✅ 複雑なエフェクト可能
- ✅ 数千の星でも滑らか

**デメリット:**
- ❌ 実装が複雑
- ❌ 学習コストが高い

**推奨用途:** Phase 2以降（500個以上の星）

#### 推奨アプローチ
```
Phase 1: SwiftUI Canvas
  - プロトタイプ
  - 100個の星
  - 基本的なグラデーション

Phase 2: Metal移行
  - パフォーマンス改善
  - 天の川追加
  - 高度なエフェクト
```

```swift
struct StarViewModel: Identifiable {
    let id: String

    let x: CGFloat          // スクリーン座標 X

struct SkyViewModel {
```

## 🎮 インタラクション設計

### 基本操作

**ドラッグ:**
- 視点の移動（方位角の変更）
- 慣性スクロール

**ピンチ:**
- ズーム（視野角の変更）
- 30° - 120° の範囲

**タップ:**
- 星の選択
- 詳細情報表示（ポップオーバー）

**ダブルタップ:**
- 星を中心に配置
- スムーズアニメーション

### 高度な機能（Phase 2）

**時間操作:**
- スライダーで時刻変更
- 再生ボタンで時間経過アニメーション
- 速度調整（1x, 10x, 100x, 1000x）

**ARモード:**
- iPadのジャイロを使用
- 実際の方角に星を表示
- CoreLocationと連携
